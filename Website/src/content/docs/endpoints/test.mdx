## Examples

<Tabs>
<TabItem label="cURL">
```bash
curl http://your-api-host/test
```
</TabItem>

<TabItem label="HTTP Request String">
```http
GET /test HTTP/1.1
Host: your-api-host
```
</TabItem>

<TabItem label="Fetch (JS)">
```js
// my-test-file.js
const apiUrl = 'http://your-api-host/test';

console.log(response);
```
</TabItem>
</Tabs>

## Response

### Body

The response will be a simple text string indicating the functionality of the API.

**Example:**
```plaintext
The API is working!
```

## Error Handling

In case of an error, the response will include an error message.

<Tabs>
<TabItem label="Invalid URL">
This error is returned when an invalid URL is accessed.

```plaintext
Invalid URL!
```
</TabItem>
</Tabs>

This endpoint is primarily used for testing purposes and provides a quick confirmation of the API's operational status.
